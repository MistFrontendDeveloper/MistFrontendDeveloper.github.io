✅ React 中类组件和函数组件的区别是什么？

  1. 心智模型：类组件基于面向对象，而函数组件基于函数式编程的。
  2. 逻辑复用：类组件的逻辑难以复用，而函数组件的逻辑可以通过 Hooks 复用。
  3. 状态管理：类组件通过 this.state 和 this.setState 来管理状态，而函数组件自身不能管理状态。
  4. 生命周期：类组件有生命周期，而函数组件没有生命周期。
  5. 代码量：类组件存在大量模版代码，因此代码量比函数组件大。

✅ React 从 Mixin 到 HOC 再到 Hooks 的发展历程中各自的优缺点是什么？

Mixin 优点：

  • 逻辑复用

Mixin 缺点：

  • 命名冲突
  • 隐式依赖

HOC 优点：

  • 逻辑复用
  • 显式包装（Fix Mixin：命名冲突、隐式依赖）

HOC 缺点：

  • 嵌套地狱

Hooks 优点：

  • 逻辑复用
  • 避免嵌套（Fix HOC：嵌套地狱）

Hooks 缺点：

  • 学习曲线
  • 语法限制

✅ 为什么 setState 是异步的？

React 为了提高渲染性能，默认会对 setState 进行批处理，如果多次调用 setState，React 只会保留最后一次调用。

✅ React 的 setState 是同步的还是异步的？

在 React 18 之前：

在 React 控制流内，setState 是异步的，但在 React 控制流外，setState 是同步的。

ps：React 生命周期和 React 事件处理函数等代码在 React 控制流内，而原生事件和 setTimeout 等代码不在。

在 React 18 之后：

setState 总是异步的。

✅ React 实现组件通信的方法？

父子组件通信：

  • props
  • useImperativeHandle

兄弟组件通信：

  • 状态提升
  • 全局状态管理器（Redux、Mobx 等）

✅ 什么是合成事件？

合成事件是一种对原生 DOM 事件的封装，作用是为了解决跨浏览器的兼容性问题。

✅ React 的性能优化方法？

  • 使用 key。
  • 使用 shouldComponentUpdate、PureComponent、memo。
  • 使用 immutable。
  • 使用 useMemo、useCallback。
  • 使用 Suspense 和 lazy。

✅ React 的生命周期？

  • constructor
  • getDerivedStateFromProps
  • render
  • componentDidMount
  • shouldComponentUpdate
  • getSnapshotBeforeUpdate
  • componentDidUpdate
  • componentWillUnmount

✅ 什么是虚拟 DOM？

虚拟 DOM 是 JS 对象对真实 DOM 的抽象，作用是为了提高渲染性能。

✅ 虚拟 DOM 的优缺点是什么？

优点：

  • 性能提升（大多数情况下）
  • 跨平台支持
  • 简化 DOM 操作

缺点：

  • 性能下降（极少数情况下）
  • 内存消耗
