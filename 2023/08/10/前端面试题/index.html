✅ setTimeout 的工作原理是什么？

到达指定的延迟时间后，setTimeout会将回调函数添加到消息队列中，当调用栈为空时，消息队列中的回调函数会按照先进先出的原则被依次执行。

✅ JavaScript 中 0.1 + 0.2 为什么不等于 0.3？如何解决？

JavaScript 中的数字都是以 64 位浮点数的形式存储的，而 0.1 和 0.2 的二进制表示都是无限循环的，因此在计算机中无法精确表示，所以 0.1 + 0.2 不等于 0.3。

解决方法：

方法一：转换为整数处理

方法二：使用 toFixed()

✅ 什么是防抖和节流？

防抖：

防抖是指在事件被触发 n 秒后再执行回调函数，如果在这 n 秒内又被触发，则重新计时。

节流：

节流是指在事件被触发 n 秒后执行回调函数，如果在这 n 秒内又被触发，则忽略该事件。

✅ 防抖和节流的区别是什么？

防抖是等待事件停止触发后执行一次处理，而节流是按固定时间间隔执行处理。

React

❓ 什么是 React？

暂无

✅ React 的优缺点是什么?

优点：

跨平台、虚拟 DOM、单向数据流、JSX、组件化、函数式编程、服务端渲染、社区活跃等。

缺点：

学习成本高、过于灵活的架构等。

✅ React 和 Vue 的区别是什么？

  1. 学习成本：React 的学习成本比 Vue 高。
  2. 数据流：React 使用单向数据流，而 Vue 使用双向数据绑定。
  3. 社区支持：React 的社区支持比 Vue 好。

✅ 什么是受控组件和非受控组件？

受控组件：

受控组件是指表单状态由 React 状态来管理的组件。

非受控组件：

非受控组件是指表单状态由 DOM 自身来管理的组件。

✅ 类组件和函数组件的区别是什么？

  1. 心智模型：类组件基于面向对象，而函数组件基于函数式编程的。
  2. 逻辑复用：类组件的逻辑难以复用，而函数组件的逻辑可以通过 Hooks 复用。
  3. 状态管理：类组件通过 this.state 和 this.setState 来管理状态，而函数组件自身不能管理状态。
  4. 生命周期：类组件有生命周期，而函数组件没有生命周期。
  5. 代码量：类组件存在大量模版代码，因此代码量比函数组件大。

✅ 从 Mixin 到 HOC 再到 Hooks 的发展历程中各自的优缺点是什么？

Mixin 优点：

  • 逻辑复用

Mixin 缺点：

  • 命名冲突
  • 隐式依赖

HOC 优点：

  • 逻辑复用
  • 显式包装（Fix Mixin：命名冲突、隐式依赖）

HOC 缺点：

  • 嵌套地狱

Hooks 优点：

  • 逻辑复用
  • 避免嵌套（Fix HOC：嵌套地狱）

Hooks 缺点：

  • 学习曲线
  • 语法限制

✅ 为什么 setState 是异步的？

React 为了提高渲染性能，默认会对 setState 进行批处理，如果多次调用 setState，React 只会保留最后一次调用。

✅ React 的 setState 是同步的还是异步的？

在 React 18 之前：

在 React 控制流内，setState 是异步的，但在 React 控制流外，setState 是同步的。

ps：React 生命周期和 React 事件处理函数等代码在 React 控制流内，而原生事件和 setTimeout 等代码不在。

在 React 18 之后：

setState 总是异步的。

浏览器与网络

✅ 什么是事件流？

事件流指在网页中，事件从发生到处理的过程，分为捕获、目标和冒泡三个阶段。

✅ 什么是事件委托？作用是什么？

事件委托是一种通过将事件处理程序绑定到父元素来代理处理其子元素事件的技术，优势在于减少处理程序数量、支持动态元素、提高性能和简化代码。

✅ Get 请求和 Post 请求的区别是什么？

  • 使用场景：Get 用于获取数据，而 Post 用于提交数据。
  • 请求参数位置：Get 请求的参数会附加在 URL 上，而 Post 请求的参数会放在请求体中。
  • 安全性：由于 Get 请求的参数在 URL 上，相较于 Post 请求更不安全。
  • 浏览器缓存：Get 请求的参数会被浏览器缓存，而 Post 请求的参数不会被浏览器缓存。
  • 请求参数类型限制：Get 请求的参数只能是字符串，而 Post 请求的参数可以是更多类型。

✅ 什么是跨域？如何解决？

跨域是指在网页中，由于浏览器的安全策略限制，JavaScript 代码无法直接访问不同源（域名、协议、端口）的资源。

解决方法：

CORS、JSONP、代理服务器等。

✅ 什么是 XSS 攻击 和 CSRF 攻击？如何防范？

XSS 攻击：

中文意思是跨站脚本攻击，攻击者通过在网页中插入恶意脚本，使其在用户浏览时执行，从而盗取用户信息或执行恶意操作。

防范措施：

输入验证、输出编码、使用 CSP（内容安全策略）等。

CSRF 攻击：

中文意思是跨站请求伪造，攻击者通过伪装的请求，利用用户已登录的身份执行非授权操作。

防范措施：

使用随机令牌、检查 Referer 头、双重身份验证等。

✅ 什么是重绘和回流？如何避免？

重绘是指当 DOM 元素样式改变时，浏览器重新绘制元素，但是元素的几何属性没有改变，如改变颜色。

回流是指当 DOM 元素的几何属性改变时，浏览器重新计算元素的几何属性，重新布局页面，如改变元素的宽高。

避免方法：

  • 样式批处理
  • 脱离文档流
  • ”离线处理“

TypeScript

✅ 什么是 TypeScript？

TypeScript 是 JavaScript 的超集，作用是为 JavaScript 提供静态类型系统。

✅ TypeScript 的优缺点是什么？

优点：

  • 静态类型
  • 前置错误
  • 易于重构

缺点：

  • 学习成本
  • 开发成本
  • 编译成本

✅ TypeScript 的特性有哪些？

  • 接口
  • 别名
  • 类型注解
  • 泛型
  • 枚举
  • 断言
  • 命名空间
  • 声明文件
  • 工具函数
  • keyof

✅ interface 和 type 的区别是什么？

合并声明：

  • interface可以合并声明，多个同名的接口声明会自动合并成一个接口。
  • type不支持合并声明，多个同名的类型声明会报错。

可扩展性：

  • interface可以通过 extends 关键字实现继承。
  • type可以通过 & 关键字实现交叉类型。

类型描述：

  • interface适合描述对象类型。
  • type适合描述复杂类型，比如枚举类型、联合类型、交叉类型等。
