✅ 什么是原型链？

原型链是 JavaScript 中的一个概念，它是由对象的原型（prototype）组成的链式结构。当访问一个对象的属性或方法时，如果对象本身没有找到，JavaScript 会在其原型链上继续查找，直到找到该属性或方法或者到达原型链的顶端（通常是 Object.prototype）为止。

✅ 什么是防抖和节流？区别是什么？

防抖：

防抖是指在事件被触发 n 秒后再执行回调函数，如果在这 n 秒内又被触发，则重新计时。

节流：

节流是指在事件被触发 n 秒后执行回调函数，如果在这 n 秒内又被触发，则忽略该事件。

区别：

重新计时：防抖在计时期间触发会导致重新计时，而节流不会。

✅ 类组件和函数组件的区别是什么？

  1. 心智模型：类组件基于面向对象，而函数组件基于函数式编程的。
  2. 逻辑复用：类组件的逻辑难以复用，而函数组件的逻辑可以通过 Hooks 复用。
  3. 状态管理：类组件通过 this.state 和 this.setState 来管理状态，而函数组件自身不能管理状态。
  4. 生命周期：类组件有生命周期，而函数组件没有生命周期。
  5. 代码量：类组件存在大量模版代码，因此代码量比函数组件大。

✅ 从 Mixin 到 HOC 再到 Hooks 的发展历程中各自的优缺点是什么？

Mixin 优点：

  • 逻辑复用

Mixin 缺点：

  • 命名冲突
  • 隐式依赖

HOC 优点：

  • 逻辑复用
  • 显式包装（Fix Mixin：命名冲突、隐式依赖）

HOC 缺点：

  • 嵌套地狱

Hooks 优点：

  • 逻辑复用
  • 避免嵌套（Fix HOC：嵌套地狱）

Hooks 缺点：

  • 学习曲线
  • 语法限制

✅ 为什么 setState 是异步的？

React 为了提高渲染性能，默认会对 setState 进行批处理，如果多次调用 setState，React 只会保留最后一次调用。

✅ React 的 setState 是同步的还是异步的？

在 React 18 之前：

在 React 控制流内，setState 是异步的，但在 React 控制流外，setState 是同步的。

ps：React 生命周期和 React 事件处理函数等代码在 React 控制流内，而原生事件和 setTimeout 等代码不在。

在 React 18 之后：

setState 总是异步的。

浏览器与网络

✅ Get 和 Post 的区别？

  • 使用场景：Get 用于获取数据，而 Post 用于提交数据。
  • 请求参数位置：Get 请求的参数会附加在 URL 上，而 Post 请求的参数会放在请求体中。
  • 安全性：由于 Get 请求的参数在 URL 上，相较于 Post 请求更不安全。
  • 浏览器缓存：Get 请求的参数会被浏览器缓存，而 Post 请求的参数不会被浏览器缓存。
  • 请求参数类型限制：Get 请求的参数只能是字符串，而 Post 请求的参数可以是更多类型。

TypeScript

✅ 什么是 TypeScript？

TypeScript 是 JavaScript 的超集，作用是为 JavaScript 提供静态类型系统。

✅ TypeScript 的优缺点是什么？

优点：

  • 静态类型
  • 前置错误
  • 易于重构

缺点：

  • 学习成本
  • 开发成本
  • 编译成本

✅ TypeScript 有哪些特性？

  • 接口
  • 别名
  • 类型注解
  • 泛型
  • 枚举
  • 断言
  • 命名空间
  • 声明文件
  • 工具函数
  • keyof

✅ interface 和 type 的区别是什么？

合并声明：

  • interface可以合并声明，多个同名的接口声明会自动合并成一个接口。
  • type不支持合并声明，多个同名的类型声明会报错。

可扩展性：

  • interface可以通过 extends 关键字实现继承。
  • type可以通过 & 关键字实现交叉类型。

类型描述：

  • interface适合描述对象类型。
  • type适合描述复杂类型，比如枚举类型、联合类型、交叉类型等。
